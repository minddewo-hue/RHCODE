# Gemma 31B 桌面端验证记录

验证日期：2026-07-16

## 验证目标

通过 RHZYCODE 桌面端和本地模型中转，使用 Gemma 31B 开发一个“A股算力板块趋势研判助手”，同时检查模型切换、长任务、任务中断、界面状态和模型目录是否存在软件缺陷。

## 模型与结果

- 可用模型 ID：`vllm/gemma-4-31b-it-uncensored-bf16`
- 另一模型 ID：`vllm/gemma-4-31b-it-uncensored`
- BF16 模型曾成功完成一次短文本请求，说明路由和鉴权有效。
- 长编码任务出现长时间无输出、超时和上游 `502`；非 BF16 版本也返回过 `502`。
- 结论：桌面端已经能识别和选择 31B 模型，但当前中转或上游不足以稳定完成连续编码任务。不能把长任务失败判定为桌面端模型切换失败。

## 测试项目

项目目录：`validation/a-share-compute-assistant`

项目提供确定性的算力板块演示数据、趋势评分、均线、动量、RSI、量比、市场宽度、回撤、波动率、风险提示、成分股筛选、CSV 导入导出以及桌面/手机响应式界面。它只用于软件验证和研究展示，不构成投资建议。

启动命令：

```powershell
cd D:\work_space\test\validation\a-share-compute-assistant
npm start
```

默认地址：`http://127.0.0.1:4178`

## 本次发现并修复的问题

- 桌面自动化脚本仍使用旧项目选择器，无法进入项目。
- 自动化脚本没有创建新任务，容易误把旧对话状态当成新结果。
- 中断任务后，时间线项目可能一直显示为运行中。
- 已被上游移除的 26B 模型仍残留在静态目录中。
- Faker 上游超时过长，模型异常时会让任务长时间停留在运行状态。

- 算力助手：11 项测试通过，桌面和 390 px 手机视口的 Playwright 界面检查通过。
- 桌面端：类型检查通过，34 项测试通过。
- 移动端：类型检查通过，26 项测试通过。

- 模型网关：18 项测试通过。

## 后续复测建议

上游恢复稳定后，用相同模型先执行短请求，再执行 3 至 5 分钟的文件创建任务。若短请求成功、长请求仍无首字节输出，应优先检查模型服务队列、显存和中转超时；若模型选择器与实际请求模型不一致，再检查桌面端路由。
