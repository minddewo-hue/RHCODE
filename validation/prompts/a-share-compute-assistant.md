请在当前空目录中完整开发一个可运行、可测试的中文 Web 应用，名称为“算力趋势研判台”，用于分析 A 股算力板块的历史走势和趋势状态。这是 RHZYCODE 桌面端的端到端压力测试，请你直接使用工具创建代码、运行测试、修复问题，直到全部验收通过，不要只给方案。

产品边界：

- 这是研究辅助工具，不是荐股软件。页面明确标注“仅供研究，不构成投资建议”，不得输出买入、卖出、目标价或收益承诺。
- 不依赖互联网接口，不伪装成实时行情。内置数据必须明确标注为“演示数据”，并允许用户导入自己的 CSV。
- CSV 至少支持字段：date, code, name, close, change_pct, volume, amount。错误列、无效日期、非数字、重复记录和空文件必须给出可理解的中文错误。

核心功能：

- 提供一份至少 120 个交易日、8 只算力产业链演示股票的数据，覆盖服务器、光模块、液冷、IDC、算力租赁等细分方向。
- 以等权归一化方式生成板块指数，展示最近一个交易日、区间涨跌、20 日最大回撤、年化波动率。
- 计算 MA5/MA10/MA20/MA60、5 日与 20 日动量、RSI14、5/20 日量能比、站上 MA20 的个股比例、上涨家数比例。
- 使用透明、可复现的规则把趋势分为“偏强 / 震荡 / 偏弱”，显示 0-100 趋势分、规则贡献明细和风险信号。不得使用随机结论。
- 风险信号至少覆盖：趋势与量能背离、板块上涨但宽度下降、20 日回撤过大、波动率异常。
- 提供个股贡献表，可按 20 日动量、波动率、成交额、相对强弱排序，并支持细分方向筛选。
- 提供可交互的板块走势图：板块指数与均线开关、时间窗口切换、鼠标悬停显示日期和数值。可使用 Canvas，不要引入在线 CDN。
- 提供 CSV 导入、恢复演示数据、导出演示 CSV、指标说明抽屉。刷新页面后记住时间窗口、均线开关和细分筛选。

界面要求：

- 第一屏就是可操作的研究工作台，不做营销落地页。
- 桌面端信息密度适中，适合反复扫描比较；移动端必须能完整使用且无横向溢出。
- 使用克制的中性背景，涨跌颜色只用于数据语义；避免大圆角、渐变、装饰性卡片堆叠和超大标题。
- 所有按钮、筛选、空状态、错误状态、导入状态都必须完整可用。文字不能重叠或被截断。

工程要求：

- 只使用当前机器已有的 Node.js 和浏览器能力，禁止依赖在线服务；尽量不增加第三方依赖。
- 将纯计算逻辑放入独立模块，浏览器 UI 与核心分析不得重复实现算法。
- 提供本地静态服务器并防止目录穿越；`npm start` 启动，默认监听 127.0.0.1，可通过 PORT 设置端口。
- 使用 Node 内置测试框架编写测试，至少覆盖：CSV 解析、指标边界、趋势分类、风险信号、排序筛选、服务器目录穿越。`npm test` 必须通过。
- 提供 README，写清数据口径、启动方法、CSV 格式、指标规则、限制和免责声明。

完成步骤：先实现，再运行 `npm test`，对所有 JS 文件执行语法检查，启动服务器并用 HTTP 验证首页与主要静态资源。发现任何错误都要自行修复。最终只汇报创建内容、测试结果、已知限制和运行地址。
